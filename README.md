# TeamLab

TeamLab — платформа для поиска и формирования команд под pet-, учебные и некоммерческие проекты.

Позволяет:
- собрать команду под идею
- найти проект для участия
- получить практический опыт и кейсы

Основной сценарий:
проект → роли → отклики → команда
---
## 1. Название и описание

TeamLab — веб-платформа для подбора и формирования команд под проекты. Она предназначена для владельцев проектов, которым нужно заполнить роли в своих командах, и для специалистов, ищущих интересные проектные роли. TeamLab управляет полным циклом подбора: от публикации проекта и создания ролей до получения откликов пользователей и подтверждения их участия в проекте.

Ключевая задача TeamLab — связать роли проекта и подходящих людей через управляемый процесс заявок (откликов) и принятия решений. Платформа предоставляет удобный интерфейс для создания проектов, управления ролями и обработки заявок от участников.

## 2. Основная идея

- Платформа для формирования команд под проекты.  
- Ключевая механика: создание проекта → добавление ролей → отклики пользователей на роли (RoleInterest) → подтверждение участия в команде (ProjectMembership).

## 3. Возможности (MVP)

- Создание и управление проектами.  
- Определение ролей в проекте (ProjectRole).  
- Отклики пользователей на роли (RoleInterest).  
- Подтверждение участия в проекте и управление командой (ProjectMembership).  
- Профиль пользователя с навыками и портфолио.  
- Список избранных проектов и кандидатов (функция избранного).  
- Внутренние уведомления в приложении (in-app уведомления).

## 4. Технологии

- Python 3  
- Django  
- Django REST Framework  
- JWT-аутентификация для API  
- Реляционная база данных (например, PostgreSQL)

## 5. Быстрый старт

Минимальный запуск backend-сервера:

1. Склонируйте репозиторий: `git clone <URL>`.  
2. Перейдите в директорию проекта и установите зависимости: `pip install -r requirements.txt`.  
3. Настройте переменные окружения (например, секретные ключи, настройки базы данных).  
4. Выполните миграции: `python manage.py migrate`.  
5. Запустите сервер разработки: `python manage.py runserver`.

## 6. Архитектура (кратко)

- Один объект `User` с полем `account_type` (`owner` или `participant`), определяющим сценарий пользователя.  
- Основной поток: создание проекта → добавление ролей → получение откликов (RoleInterest) → подтверждение участия в проекте (ProjectMembership).  
- Участие (`ProjectMembership`) появляется только после принятия отклика. Прямое создание членства запрещено.  
- В MVP запрещены повторные отклики на одну роль. Закрытые проекты и роли не принимают новых заявок.

## 8. API

- OpenAPI-спецификация находится в файле `teamlab_openapi-schema.yml` (или аналогичном `.yaml`) в репозитории.  
- Эту схему можно подключить к Swagger UI или другим инструментам (Postman и т.п.) для просмотра и тестирования API.
